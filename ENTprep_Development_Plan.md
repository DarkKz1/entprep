# ENTprep — План развития и путь к полноценному приложению

---

## 📍 Текущее состояние (v4.0)

**ENTprep** — React-компонент (~810 строк), работающий как артифакт или встроенный виджет. Текущие возможности:

- 3 предмета: Мат. грамотность (150 вопросов), Грамотность чтения (20 текстов × 5 = 100), История Казахстана (80 вопросов)
- Калькулятор грантов (20 вузов Алматы и Астаны)
- Таймер, перемешивание, объяснения к каждому ответу
- Прогресс и статистика (в оперативной памяти — теряется при закрытии)
- Тёмная тема, мобильный дизайн (max-width 480px)

**Ключевые ограничения:** нет персистентности данных, нет бэкенда, нет 2 профильных предметов, нет авторизации, всё в одном файле.

---

## 🗺️ Дорожная карта развития

### Фаза 1 — Стабилизация и контент (2–3 недели)

**Цель:** довести контент до минимально конкурентоспособного уровня.

**1.1 Добавить профильные предметы**  
ЕНТ включает 2 профильных предмета по 40 заданий. Нужно добавить хотя бы 4–6 популярных комбинаций: Математика + Физика, Биология + Химия, География + Английский язык, Всемирная история + Право. Для каждого — минимум 100 вопросов.

**1.2 Расширить банк вопросов**  
Текущие 330 вопросов — хороший старт, но для полноценной подготовки нужно 1 500+ по каждому предмету. На этом этапе можно использовать AI для генерации вопросов с последующей ручной проверкой.

**1.3 Сохранение прогресса**  
Перейти на localStorage / AsyncStorage, чтобы прогресс сохранялся между сессиями. Структура: история тестов, слабые темы, дата последнего входа.

**1.4 Рефакторинг кода**  
Разбить монолитный файл на модули: компоненты (Home, Test, Calc, Progress, Settings), данные (вопросы вынести в JSON-файлы), утилиты (таймер, перемешивание, расчёты). Перейти на TypeScript с типизацией.

---

### Фаза 2 — UX и умная подготовка (3–4 недели)

**Цель:** превратить тренажёр в «умного репетитора».

**2.1 Адаптивная система обучения**  
Отслеживать, какие темы пользователь решает плохо, и предлагать больше заданий по слабым сторонам. Хранить статистику по каждому вопросу: сколько раз показывался, сколько раз ошибка.

**2.2 Режим полного ЕНТ**  
Симуляция реального экзамена: 120 заданий, 5 предметов, 4 часа, автоматический подсчёт баллов по шкале ЕНТ, с порогами по каждому предмету.

**2.3 Улучшения интерфейса**  
Онбординг для новых пользователей, выбор профильных предметов при первом запуске, карточки с формулами и шпаргалки, закладки для сложных вопросов, тёмная/светлая тема.

**2.4 Расширить калькулятор грантов**  
Добавить все вузы Казахстана (не только Алматы и Астана), фильтрацию по специальности, ссылки на сайты приёмных комиссий, данные по проходным баллам за прошлые годы.

---

### Фаза 3 — Бэкенд и авторизация (4–6 недель)

**Цель:** облачная синхронизация и аналитика.

**3.1 Бэкенд**  
Supabase или Firebase как BaaS (Backend as a Service) — минимум кода, максимум функций. Хранение прогресса в облаке, синхронизация между устройствами.

**3.2 Авторизация**  
Вход через Google / Apple / Telegram — без необходимости вводить пароли. Профиль пользователя: имя, город, целевой вуз, профильные предметы.

**3.3 Аналитика и рейтинги**  
Общий рейтинг среди пользователей (анонимный), статистика по регионам, еженедельные отчёты прогресса.

**3.4 Система уведомлений**  
Push-напоминания: «Ты не занимался 2 дня», «До ЕНТ осталось N дней», ежедневный вопрос дня.

---

### Фаза 4 — AI-интеграция (2–4 недели)

**Цель:** использовать AI для персонализации.

**4.1 AI-объяснения**  
Если пользователь ошибся — кнопка «Объясни подробнее», которая через API Claude или GPT даёт развёрнутое объяснение с примерами, подстраиваясь под уровень ученика.

**4.2 Генерация тестов**  
AI генерирует новые вопросы по слабым темам, чтобы банк вопросов не повторялся.

**4.3 AI-чат-репетитор**  
Встроенный чат, где можно задать вопрос по любой теме ЕНТ и получить ответ с решением.

---

### Фаза 5 — Монетизация (параллельно с Фазой 3–4)

**Бесплатный уровень:** 30 вопросов/день, базовая статистика, 2 предмета.  
**Премиум (990–1 990 тг/мес):** безлимитные вопросы, все предметы, AI-объяснения, полная симуляция ЕНТ, офлайн-режим.  
**Другие варианты:** разовая покупка (4 990 тг), промокоды для школ, партнёрства с вузами.

## 📱 Путь к скачиваемому приложению

### Вариант 1 — React Native / Expo (рекомендуется)

**Почему:** один код для iOS и Android, быстрый старт, огромное сообщество.

**Шаги:**

1. Установить Expo CLI: `npx create-expo-app entprep`
2. Перенести логику — вопросы, тесты, расчёты переносятся почти без изменений
3. Заменить HTML-элементы на React Native компоненты: `<div>` → `<View>`, `<span>` → `<Text>`, `<button>` → `<TouchableOpacity>`, inline styles → StyleSheet.create()
4. Навигация — установить `@react-navigation/native` вместо ручного переключения экранов
5. Хранение данных — `@react-native-async-storage/async-storage` вместо localStorage
6. Публикация — Expo EAS Build позволяет собрать .apk / .ipa без Mac

**Стоимость публикации:** Google Play — $25 (разово), Apple App Store — $99/год.

### Вариант 2 — PWA (Progressive Web App)

**Почему:** не нужен магазин приложений, работает офлайн, самый быстрый путь.

**Шаги:**

1. Обернуть текущий React-код в Vite/Next.js проект
2. Добавить `manifest.json` (иконки, название, цвета)
3. Добавить Service Worker для офлайн-кеширования (workbox)
4. Пользователи смогут «Добавить на главный экран» прямо из браузера

**Плюсы:** нет комиссии магазинов, мгновенные обновления, работает на всех устройствах.  
**Минусы:** нет push-уведомлений на iOS (ограничено), нет доступа к нативным функциям, не виден в магазинах.

### Вариант 3 — Capacitor (гибрид)

**Почему:** оборачивает веб-приложение в нативную оболочку для публикации в магазинах.

**Шаги:**

1. Собрать React-приложение через Vite: `npm run build`
2. `npm install @capacitor/core @capacitor/cli`
3. `npx cap init` → `npx cap add android` → `npx cap add ios`
4. `npx cap sync` → открыть в Android Studio / Xcode → собрать

**Плюсы:** минимальные изменения в коде, доступ к нативным API через плагины.  
**Минусы:** менее отзывчивый UI чем React Native.

## 🏗️ Рекомендуемый технический стек

| Слой | Технология | Причина |
|------|-----------|---------|
| Фронтенд (веб) | React + Vite + TypeScript | Скорость разработки, типизация |
| Мобильное приложение | Expo (React Native) | Один код, быстрая сборка |
| Бэкенд | Supabase | Бесплатный тир, auth, БД, realtime |
| БД | PostgreSQL (Supabase) | Надёжность, бесплатно |
| AI | Claude API (Anthropic) | Лучшие объяснения на русском |
| Аналитика | PostHog (self-hosted) | Бесплатно, GDPR-compliant |
| CI/CD | GitHub Actions + Expo EAS | Автоматическая сборка |
